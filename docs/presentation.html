Flight Route Conflict Scanner

Assessing great-circle routes against conflict data, NOTAMs, and airline behaviour

Live app: run npm run dev from repo root — UI at Vite’s port (often 5173), API at 3001.

Context

Why this exists

  • Operations and risk teams need a fast, explainable view of how a passenger route relates to hotspots — not just a line on a map.
  • Conflict activity, airspace notices, and recent airline deviations can signal elevated cancellation or reroute risk.
  • FRCS ties those signals together in one scanner workflow.
Capabilities

What the application shows

  • Route scan — origin/destination, ranked route options, map with polylines and nearby zones.
  • Risk dashboard — weighted factors (conflict proximity, rerouting signals, NOTAMs, historical baseline).
  • Live advisories sidebar — stream of NOTAM-style notices.
  • Airlines & Anomalies tabs — profiles and anomaly list for the scanned city-pair.
Under the hood

Architecture (high level)

  • Frontend — Lit components, Leaflet map, shared TypeScript types.
  • Backend — Fastify REST + SQLite (seeded conflict events, zones, NOTAMs, flight history).
  • Data — npm run seed loads representative airports and sample conflict/airspace data (not production feeds in this repo).
Live demo — step 1

Start the stack

Terminal: cd repo root → npm install (once) → npm run seed → npm run dev.

Open the URL Vite prints. Confirm the advisories sidebar loads (proves API proxy + backend).

Live demo — step 2

Scan a seeded route

Use only airports that exist after seed (subset of global hubs). Examples:

  • JFK → LHR (transatlantic)
  • TLV → DXB (regional)
  • BOS → CDG (if BOS is in your seed list)

Type codes or pick from suggestions, then Scan Route.

Live demo — step 3

Walk through the results

  • Map — multiple ranked paths; select a row to change highlighted route and zone circles.
  • Route cards — score, distance, reasoning, nearest conflict zones.
  • Risk dashboard — read the summary and factor breakdown aloud.
  • Alternatives — shown when the best route’s safety score is below the configured threshold.
Live demo — step 4

Airlines & anomalies

  • Switch to Airlines — brief profiles tied to the scanned route label.
  • Switch to Anomalies — list driven by the same route context.
  • Return to Scanner for another city-pair to show repeatability.
Honest scope

What this prototype is not

  • Not a certified flight-planning or dispatch tool.
  • Seeded / sample data — not live ACLED, Eurocontrol, or airline FOQA.
  • Airport coverage is limited to the seed list until you expand data ingestion.
